## 4. 启动可行性评估 (Startup & Running Capability)

> [!IMPORTANT]
> **结论：当前数据库数据完全支持项目正常启动，且能生成具有高度参考价值的“冲、稳、保、垫”推荐报告。系统具备完善的优雅降级 (Fallback) 机制。**

### 4.1 为什么缺少招生计划也能运行？
在 `src/server/gaokao/gaokao_recommend.js` 中，推荐引擎的检索与召回逻辑设计非常鲁棒：
1.  **数据基准降级**：
    当用户输入分数（系统通过 2026 年一段表查到位次）后，推荐算法**完全依赖 `gaokao_admission_records` 里的 4.5 万条历史录取数据**进行位次比对。
2.  **位次差计算 (`rankGap`)**：
    算法根据 2024、2025 两年的平均录取最低位次（`avgMinRank`）与考生的当前位次作差，自动划分出“冲 (rush)”、“稳 (stable)”、“保 (safe)”、“垫 (fallback)”四个梯度并排序。
3.  **无计划下的安全提示**：
    即使 `plans` (当年招生计划) 的数量为 0，系统只会将 `sufficientForProbability` 标记为 `false`，并在 AI 报告生成的 `risks`（风险提示）和 `nextDataNeeded`（缺失数据）中注明：“需手动核对当年招生简章计划、选科限制及体检要求”，而**完全不会导致 API 崩溃或页面无法渲染**。

## 5. 缺失数据补充方案 (Data Import Strategy)

如果在后续的运营中，您拿到了 2026 年最新的“招生计划”CSV 文件，可以通过如下步骤导入：

### 5.1 招生计划 CSV 导入命令
整理好符合列头（如 `院校名称`、`专业名称`、`招生人数`、`选科要求` 等）的 CSV 文件后，在根目录下执行：
```bash
node scripts/gaokao/import-csv.mjs --table=plans --file=path/to/your_plans.csv --province=浙江 --year=2026
```
该脚本会自动匹配并填充 `gaokao_enrollment_plans` 表，数据库一旦检测到 plans count > 0，降级提示就会自动消除，升级为 100% 完美的精准概率推荐。
